Organizing the code

  • Code is chunked into bite sized modules using RequireJS
  • Each module is separate file
    • Easy code reuse
    • Can help in testability if done right
    • Have to keep less in your head at once
  • RequireJS has an optimizer (rjs) that can compile all your modules into one file for production

Organizing the code - responsibilities

  • MVC / MVWhatever
    • Model - Data
    • View - Filtered presentation of data
    • Controller - User interaction / glue

Organizing the code - Backbone

  • Very flexible
  • Has a good pub/sub implementation
    • Listen to models
    • Create channels to use as event buses
      // Create arbitrary channels
var channels = { oneChannel: {}, otherChannel: {} };

// Mix in events
_.forEach(channels, function(aChannel) {
    _.extend(app.channels[aChannel],
        Backbone.Events);
}

Organizing the code - Backbone

  • Since Backbone is so bare bones, there is a lot of boiler plate
  • Can write your own time savers or use things like Marionette
